SQL

Key Outcomes

  • Gained hands-on experience working with relational databases using MySQL Workbench.
  • Designed and created normalised databases to ensure data integrity and minimise redundancy (up to 3rd Normal Form).
  • Wrote efficient SQL queries to retrieve and analyse key business information for real-world scenarios.
  • Developed proficiency in using JOINs (INNER, LEFT, RIGHT, FULL) to combine data across multiple tables.
  • Performed data aggregation and filtering using GROUP BY, HAVING, WHERE, and ORDER BY clauses for insightful reporting.
  • Created views and stored procedures to simplify complex queries and improve database efficiency.
  • Gained experience optimising performance using primary keys, foreign keys, and indexing strategies.
  • Final solutions supported effective data-driven decision making through comprehensive reporting and analysis.

Skills & Tools Utilised

  • MySQL Workbench: Designed, developed, and managed relational databases using a graphical interface and SQL scripts.
  • Database Design & Normalisation: Applied normalisation principles up to Third Normal Form (3NF) to reduce data redundancy and improve integrity.
  • Primary & Foreign Keys: Defined relational structures and enforced data consistency through proper key constraints.
  • SQL Querying: Wrote complex queries using SELECT, JOINs (INNER, LEFT, RIGHT, FULL), WHERE, GROUP BY, HAVING, and ORDER BY clauses.
  • Views & Stored Procedures: Created reusable views for simplified reporting and stored procedures for automating repetitive analysis tasks.
  • Performance Optimisation: Improved query efficiency through the use of indexes and execution plan analysis.
  • Data Analysis & Reporting: Extracted actionable insights from complex datasets to support business scenarios and decision-making processes.

Reflection

This project significantly deepened my understanding of relational database management and real-world data modeling. I learned the importance of proper database normalisation to maintain clean, efficient, and scalable data structures. Through hands-on practice, I became confident in writing complex SQL queries and combining data from multiple tables using advanced joins and subqueries. One key takeaway was how the thoughtful use of views, stored procedures, and indexing can dramatically improve both development efficiency and database performance. This experience has strengthened my ability to analyse data effectively and provide valuable insights that support informed, data driven business decisions.
